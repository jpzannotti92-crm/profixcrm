1. Introducción

IATrade CRM es un sistema completo de gestión de relaciones con clientes diseñado específicamente para empresas de trading e inversión. Este manual te guiará paso a paso a través del proceso de instalación y configuración inicial del sistema.

Características principales:
  • Gestión completa de leads y clientes
  • Sistema de roles y permisos granular
  • Integración con plataformas de trading
  • Dashboard analítico en tiempo real
  • Sistema de notificaciones automáticas
  • Reportes y métricas avanzadas
Importante: Asegúrate de leer completamente este manual antes de comenzar la instalación. Un proceso de instalación incorrecto puede resultar en problemas de funcionamiento o seguridad.

2. Requisitos del Sistema

2.1 Requisitos de Servidor

Servidor Web

  • • Apache 2.4+ o Nginx 1.18+
  • • Soporte para mod_rewrite (Apache)
  • • HTTPS habilitado (recomendado)

Base de Datos

  • • MySQL 8.0+ o MariaDB 10.5+
  • • Mínimo 1GB de espacio libre
  • • Usuario con privilegios CREATE/DROP

PHP

Versión: PHP 8.1 o superior Extensiones requeridas: - pdo_mysql - mbstring - openssl - tokenizer - xml - ctype - json - bcmath - curl - fileinfo - gd - zip Configuración mínima: memory_limit = 256M max_execution_time = 300 upload_max_filesize = 50M post_max_size = 50M

2.2 Requisitos de Hardware

CPU

2+ cores, 2.0GHz+

RAM

4GB mínimo, 8GB recomendado

Almacenamiento

10GB libres, SSD recomendado

3. Preparación del Servidor

3.1 Configuración de Apache

Crea un Virtual Host para tu aplicación:

<VirtualHost *:80> ServerName tu-dominio.com DocumentRoot /var/www/html/iatrade-crm/public <Directory /var/www/html/iatrade-crm/public> AllowOverride All Require all granted </Directory> ErrorLog ${APACHE_LOG_DIR}/iatrade-error.log CustomLog ${APACHE_LOG_DIR}/iatrade-access.log combined </VirtualHost>

Habilita los módulos necesarios:

sudo a2enmod rewrite sudo a2enmod ssl sudo systemctl restart apache2

3.2 Configuración de MySQL

Crea una base de datos y usuario para la aplicación:

mysql -u root -p CREATE DATABASE iatrade_crm CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; CREATE USER 'iatrade_user'@'localhost' IDENTIFIED BY 'tu_password_seguro'; GRANT ALL PRIVILEGES ON iatrade_crm.* TO 'iatrade_user'@'localhost'; FLUSH PRIVILEGES; EXIT;

3.3 Permisos de Archivos

Configura los permisos correctos:

# Propietario de archivos sudo chown -R www-data:www-data /var/www/html/iatrade-crm # Permisos de directorios sudo find /var/www/html/iatrade-crm -type d -exec chmod 755 {} \; # Permisos de archivos sudo find /var/www/html/iatrade-crm -type f -exec chmod 644 {} \; # Permisos especiales para directorios de escritura sudo chmod -R 775 /var/www/html/iatrade-crm/storage sudo chmod -R 775 /var/www/html/iatrade-crm/bootstrap/cache

4. Proceso de Instalación

4.1 Descarga e Instalación

  1. Descarga los archivos del sistema a tu servidor
  2. Extrae el contenido en el directorio web de tu servidor
  3. Navega a http://tu-dominio.com/install
  4. Sigue el asistente de instalación paso a paso

4.2 Pasos del Asistente

Paso 1: Verificación de Requisitos

El sistema verificará automáticamente que tu servidor cumple con todos los requisitos necesarios.

Si aparecen errores: Corrige todos los problemas antes de continuar. El instalador no te permitirá avanzar hasta que todos los requisitos estén satisfechos.

Paso 2: Configuración de Base de Datos

Proporciona los datos de conexión a tu base de datos MySQL.

  • Host del servidor (generalmente 'localhost')
  • Nombre de la base de datos
  • Usuario y contraseña
  • Puerto (3306 por defecto)

Paso 3: Creación del Super Administrador

Crea la cuenta del administrador principal del sistema.

Importante: Guarda estas credenciales en un lugar seguro. Este usuario tendrá acceso completo al sistema.

Paso 4: Configuración del Sistema

Configura los parámetros básicos de la aplicación.

  • Nombre de la aplicación
  • URL base del sistema
  • Zona horaria
  • Moneda predeterminada
  • Configuración de email

Paso 5: Finalización

Verificación final y limpieza de archivos de instalación.

¡Instalación completada! El sistema estará listo para usar.

5. Configuración Post-Instalación

5.1 Configuración de Seguridad

SSL/HTTPS

Configura un certificado SSL para tu dominio:

# Con Let's Encrypt (recomendado) sudo apt install certbot python3-certbot-apache sudo certbot --apache -d tu-dominio.com

Firewall

Configura el firewall para permitir solo los puertos necesarios:

sudo ufw allow 22 # SSH sudo ufw allow 80 # HTTP sudo ufw allow 443 # HTTPS sudo ufw enable

5.2 Configuración de Cron Jobs

Configura las tareas programadas para el mantenimiento automático:

# Editar crontab sudo crontab -e # Agregar las siguientes líneas: # Limpieza de logs cada día a las 2:00 AM 0 2 * * * /usr/bin/php /var/www/html/iatrade-crm/artisan logs:clear # Backup de base de datos cada día a las 3:00 AM 0 3 * * * /usr/bin/mysqldump -u usuario -ppassword iatrade_crm > /backups/iatrade_$(date +\%Y\%m\%d).sql # Procesamiento de notificaciones cada 5 minutos */5 * * * * /usr/bin/php /var/www/html/iatrade-crm/artisan notifications:process

5.3 Configuración de Email

Configura el servidor SMTP para el envío de emails:

# En el archivo .env o configuración del sistema MAIL_MAILER=smtp MAIL_HOST=smtp.gmail.com MAIL_PORT=587 MAIL_USERNAME=tu-email@gmail.com MAIL_PASSWORD=tu-password-de-aplicacion MAIL_ENCRYPTION=tls MAIL_FROM_ADDRESS=noreply@tu-dominio.com MAIL_FROM_NAME="IATrade CRM"

6. Solución de Problemas

6.1 Problemas Comunes

Error: "No se puede conectar a la base de datos"

Causa: Credenciales incorrectas o servidor MySQL no disponible.

Solución:

  • Verifica que MySQL esté ejecutándose: sudo systemctl status mysql
  • Confirma las credenciales de la base de datos
  • Verifica que el usuario tenga los permisos necesarios

Error: "Extensión PHP no encontrada"

Causa: Falta una extensión PHP requerida.

Solución:

# Instalar extensiones PHP faltantes sudo apt update sudo apt install php8.1-mysql php8.1-mbstring php8.1-xml php8.1-curl php8.1-gd php8.1-zip sudo systemctl restart apache2

Error: "Permisos insuficientes"

Causa: Los directorios no tienen los permisos correctos.

Solución:

sudo chown -R www-data:www-data /var/www/html/iatrade-crm sudo chmod -R 755 /var/www/html/iatrade-crm sudo chmod -R 775 /var/www/html/iatrade-crm/storage

6.2 Modo de Depuración

Para obtener más información sobre errores, habilita el modo debug:

# En el archivo de configuración APP_DEBUG=true APP_ENV=local # Revisar logs de errores tail -f /var/log/apache2/error.log tail -f /var/www/html/iatrade-crm/storage/logs/laravel.log
Importante: Desactiva el modo debug en producción por seguridad.

7. Mantenimiento

7.1 Backups Regulares

Configura backups automáticos de la base de datos y archivos:

#!/bin/bash # Script de backup diario DATE=$(date +%Y%m%d_%H%M%S) BACKUP_DIR="/backups" DB_NAME="iatrade_crm" DB_USER="iatrade_user" DB_PASS="tu_password" # Backup de base de datos mysqldump -u $DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/db_backup_$DATE.sql # Backup de archivos tar -czf $BACKUP_DIR/files_backup_$DATE.tar.gz /var/www/html/iatrade-crm # Limpiar backups antiguos (más de 30 días) find $BACKUP_DIR -name "*.sql" -mtime +30 -delete find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
